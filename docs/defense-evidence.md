# AsaLingo — доказательная база для защиты на конкурсе

## Формулировка проблемы

Изучающие язык выходят на плато, когда практические материалы перестают соответствовать их реальному уровню. Готовые инструменты вроде Duolingo, Anki и Quizlet либо используют заранее заданные банки упражнений, либо опираются на механическое повторение карточек. В результате нет обратной связи между тем, что ученик уже показал на практике, и тем контентом, который он получает дальше. Для словарного запаса эта проблема еще заметнее: большинство инструментов воспринимают лексику как список для запоминания, а не как систему связанных значений на конкретном уровне владения языком.

**Тезис AsaLingo:** каждое лексическое упражнение должно генерироваться заново с учетом уровня CEFR ученика, этапа освоения письменности, слабых поднавыков, родного языка и текущей позиции в учебной дорожной карте, а не выбираться из статического банка.

**Короткий питч:** AsaLingo не просто показывает карточки. Платформа строит персональный учебный маршрут, оценивает уровень ученика, генерирует упражнения с учетом CEFR и проверяет ответы на стороне сервера.

## Целевая аудитория

- Взрослые, изучающие иностранный язык для работы, путешествий или учебы
- Ученики уровней CEFR A1-C2, которым нужен структурированный рост словарного запаса
- Изучающие языки с нелатинской письменностью: японская кана/кандзи, корейский хангыль, китайский
- Преподаватели и самостоятельные ученики, которым нужен открытый стек с возможностью самостоятельного хостинга

## Сравнение с существующими инструментами

| Возможность | AsaLingo | Duolingo | Anki | Quizlet |
|---|---|---|---|---|
| AI-сгенерированные упражнения без жестко заданных банков | ✅ | ❌ | ❌ | ❌ |
| Контент, адаптированный под CEFR A1-C2 | ✅ | Частично | ❌ | ❌ |
| Адаптивный тест определения уровня | ✅ | ❌ | ❌ | ❌ |
| Интервальное повторение с настраиваемыми интервалами | ✅ | ❌ | ✅ | ❌ |
| Локализация интерфейса на 10 языков | ✅ | ✅ | ❌ | ❌ |
| Учет этапов освоения письменности: Latin -> Kana -> Kanji | ✅ | Частично | ❌ | ❌ |
| Самостоятельный хостинг / открытый стек | ✅ | ❌ | ✅ | ❌ |

**Почему это важно для учеников:** Duolingo бесконечно повторяет знакомые предложения; Anki требует вручную создавать карточки; Quizlet дает совместные наборы карточек, но не персонализирует обучение. AsaLingo генерирует упражнения по запросу на основе профиля ученика, поэтому контент остается подходящим по мере роста навыка.

## Зачем здесь нужен AI

1. **Определение уровня без статического банка тестов.** Адаптивная сессия определения уровня генерирует задания на лету и подстраивает их под текущую оценку уровня ученика, смешивая лексику, грамматику, проверку письменности и короткие задания на продукцию речи.

2. **Разнообразие упражнений в масштабе.** Одно словарное слово может превратиться в несколько типов упражнений: multiple choice, fill-blank, sentence-writing, image-based. Все они адаптируются под CEFR-диапазон ученика и его родной язык без ручного создания материалов.

3. **Оценка открытых ответов.** Упражнения на написание предложений и короткую продукцию требуют семантической оценки, а не простого совпадения ключевых слов. AI проверяет, грамматически ли правдоподобно предложение ученика и правильно ли использовано целевое слово.

4. **Объяснение на родном языке ученика.** AI генерирует объяснения слов и примеры предложений на языке, который понимает ученик, вместо универсальных словарных статей.

5. **Поиск похожих слов.** Система показывает семантически связанную лексику, которую ученик еще не видел, расширяя его лексическую сеть, а не тренируя изолированные слова.

## Обзор архитектуры

```
Browser (React 18 + Vite)
  └── TanStack Query  ←── Axios client (auto-refresh on 401)
        └── REST API (Express 4 + TypeScript)
              ├── JWT auth (access 15m + refresh 7d, rotation)
              ├── Google OAuth (Passport)
              ├── Rate limiting (Redis / memory fallback)
              ├── Mongoose ODM → MongoDB 7
              │     ├── users, refreshtokens
              │     ├── vocabularywords, userwords
              │     ├── learningsessions
              │     └── aicontentcaches (TTL 7d)
              └── DeepSeek API (AI generation)
```

Ключевые архитектурные решения:
- **MongoDB вместо SQL:** гибкая схема для развивающихся форматов упражнений и TTL-индексы для истечения срока действия AI-кеша.
- **DeepSeek:** экономичная модель для массовой генерации контента под каждого ученика.
- **Ротация JWT refresh-токенов:** отзыв сессий и защита от повторного использования токена без полноценного OAuth-сервера.
- **Серверная проверка ответов:** сгенерированные упражнения сохраняются в сессиях, привязанных к пользователю; endpoint отправки ответа заново проверяет ответы на стороне сервера, поэтому клиент не может подменить флаг `correct`.

## Масштабируемость и roadmap разработки

### Текущий масштаб
- Stateless backend — горизонтально масштабируется за балансировщиком нагрузки
- Rate limiting поддерживает Redis для общего состояния между несколькими инстансами
- AI-кеш с TTL 7 дней снижает количество повторных API-запросов для повторяющихся поисков слов

### Ближайший roadmap
1. **Streak и расписание интервального повторения** — показывать слова, которые пора повторить, из `userwords` на основе интервалов повторения
2. **Аудиопроизношение** — интеграция TTS для упражнений на аудирование
3. **Gap-fill по медиатранскриптам** — аудирование на основе клипов с cloze-упражнениями
4. **Режим преподавателя / класса** — общие списки слов и аналитика по группам
5. **Mobile PWA** — офлайн-повторение слов через service worker

### Запас архитектуры для роста
- Шардирование MongoDB для коллекций словаря и пользовательских слов
- Redis-backed rate limiting уже подключен архитектурно
- Слой AI-кеша поглощает пики трафика для популярных словарных запросов
- Путь миграции Docker Compose → Kubernetes остается простым: stateless backend и единый MongoDB URI
