# AsaLingo — AI-Powered Vocabulary Learning Platform

A full-stack vocabulary learning platform with adaptive CEFR-level content, spaced repetition, and AI-generated explanations, examples, and quizzes.

**Docs:** [Demo setup guide](docs/DEMO.md) · [Defense evidence & architecture](docs/defense-evidence.md)

## Stack

| Layer | Technology |
|---|---|
| Frontend | React 18 + TypeScript + Vite + TanStack Query + Zustand + Tailwind CSS |
| Backend | Node.js + Express + TypeScript + Mongoose |
| Database | **MongoDB 7** |
| Auth | JWT (access + refresh tokens) + Google OAuth 2.0 |
| AI | DeepSeek |
| Infra | Docker + Docker Compose |

## Why AsaLingo?

**Competition pitch:** AsaLingo does not just show flashcards; it builds a personal learning route, estimates the learner's level, generates CEFR-aware exercises, and verifies answers server-side.

| Feature | AsaLingo | Duolingo | Anki | Quizlet |
|---|---|---|---|---|
| AI-generated exercises (no hardcoded questions) | ✅ | ❌ | ❌ | ❌ |
| CEFR-adaptive content (A1–C2) | ✅ | Partial | ❌ | ❌ |
| Spaced repetition with custom intervals | ✅ | ❌ | ✅ | ❌ |
| Adaptive placement test | ✅ | ❌ | ❌ | ❌ |
| 10-language UI localization | ✅ | ✅ | ❌ | ❌ |
| Self-hosted / open stack | ✅ | ❌ | ✅ | ❌ |
| Script progression (Latin → Kana → Kanji) | ✅ | Partial | ❌ | ❌ |

## Technical Decisions

- **MongoDB over PostgreSQL:** AsaLingo stores flexible vocabulary, roadmap, placement, and generated exercise documents that evolve quickly; MongoDB also gives us TTL indexes that fit the AI cache naturally.
- **DeepSeek for AI generation:** the app generates a high volume of explanations, exercises, and placement items, so a cost-effective model matters for keeping runtime generation practical.
- **JWT refresh rotation over fully stateless auth:** rotating refresh tokens adds state, but it gives better session revocation and replay protection for a user-facing app.

---

## Content Policy

AsaLingo must not ship ready-made tests, quiz banks, or pre-authored learner exercises in the app. Placement items, roadmap lessons, practice questions, transcript gaps, and quiz options must be generated by AI at runtime from the learner profile, roadmap context, and vocabulary available for that user.

Seed scripts may create structural learning data such as curriculum nodes and starter vocabulary, but they must not seed completed tests, media transcript exercises, or fixed answer-option sets. If AI is not configured, test/quiz generation should fail clearly instead of falling back to canned content.

## Quick Start

### 1. Clone and configure environment

```bash
cp .env.example .env
# Edit .env with your values (see Environment Variables section below)
```

### 2. Start with Docker Compose (recommended)

```bash
docker-compose up --build
```

This starts:
- **MongoDB** on port `27017`
- **Backend API** on port `4000`
- **Frontend** on port `5173`

### 3. Seed the database

```bash
docker-compose exec backend npm run db:seed
```

### 4. Open the app

- Frontend: http://localhost:5173
- Backend API: http://localhost:4000/api
- Health check: http://localhost:4000/health

## Environment Variables

Copy `.env.example` to `.env` and fill in your values:

| Variable | Required | Description |
|---|---|---|
| `MONGODB_URI` | Yes | MongoDB connection string |
| `JWT_SECRET` | Yes | Secret for access tokens |
| `JWT_REFRESH_SECRET` | Yes | Secret for refresh tokens |
| `JWT_EXPIRES_IN` | No | Access token expiry (default: `15m`) |
| `JWT_REFRESH_EXPIRES_IN` | No | Refresh token expiry (default: `7d`) |
| `GOOGLE_CLIENT_ID` | Optional | Google OAuth client ID |
| `GOOGLE_CLIENT_SECRET` | Optional | Google OAuth client secret |
| `GOOGLE_CALLBACK_URL` | Optional | Google OAuth callback URL |
| `DEEPSEEK_API_KEY` | Optional | DeepSeek API key for AI features |
| `SMTP_HOST` | No | SMTP host for password reset emails (default: `smtp.ethereal.email`) |
| `SMTP_PORT` | No | SMTP port for password reset emails (default: `587`) |
| `SMTP_USER` | Optional | SMTP username; if omitted with `SMTP_PASS`, Ethereal test SMTP is auto-created |
| `SMTP_PASS` | Optional | SMTP password; if omitted with `SMTP_USER`, Ethereal test SMTP is auto-created |
| `SMTP_FROM` | No | Sender used for password reset emails (default: `AsaLingo <noreply@asalingo.app>`) |
| `FRONTEND_URL` | Yes | Frontend origin (for CORS) |
| `VITE_API_URL` | Yes | Backend API URL for frontend |
| `BCRYPT_ROUNDS` | No | Password hashing rounds (default: `12`) |
| `TRUST_PROXY` | No | Express `trust proxy` setting (default: `1`) |
| `RATE_LIMIT_REDIS_URL` | No | Optional Redis URL for shared rate-limit storage |
| `AUTH_LOGIN_RATE_LIMIT_MAX` | No | Failed login attempts per auth window (default: `8`) |
| `AUTH_REGISTER_RATE_LIMIT_MAX` | No | Register attempts per auth window (default: `5`) |
| `AUTH_FORGOT_PASSWORD_RATE_LIMIT_MAX` | No | Forgot-password attempts per auth window (default: `5`) |
| `AUTH_RESET_PASSWORD_RATE_LIMIT_MAX` | No | Reset-password attempts per auth window (default: `5`) |

> **AI key:** The backend uses DeepSeek for AI features. Continue-lesson, roadmap-practice, and placement generation now reuse recent valid cached outputs when AI is temporarily unavailable. If no cached content exists, the API returns a clean `503` instead of leaking provider details.

> **Server-side verification:** Lesson and practice submission no longer trust any client-provided `correct` flag. Generated exercises are stored in a user-bound exercise session, and `/api/learning/submit` re-evaluates answers server-side from the canonical question data. AI-dependent open answers can return `pending_review`, which means the answer is stored but does not lower score, mastery, or roadmap completion.

> **Password reset email:** If SMTP credentials are configured, AsaLingo sends real reset emails through that SMTP server. If `SMTP_USER` / `SMTP_PASS` are missing, the backend automatically creates an Ethereal test inbox and logs the preview URL so the flow still works out of the box.

## API Routes

### Auth
| Method | Path | Description |
|---|---|---|
| POST | `/api/auth/register` | Register with email/password |
| POST | `/api/auth/login` | Login |
| POST | `/api/auth/refresh` | Refresh access token |
| POST | `/api/auth/logout` | Logout |
| GET | `/api/auth/me` | Get current user |
| GET | `/api/auth/google` | Start Google OAuth |
| GET | `/api/auth/google/callback` | Google OAuth callback |
| POST | `/api/auth/forgot-password` | Request password reset email |
| POST | `/api/auth/reset-password` | Consume reset token and set new password |

### Users
| Method | Path | Description |
|---|---|---|
| GET | `/api/users/me` | Get profile |
| PATCH | `/api/users/me` | Update profile |
| POST | `/api/users/me/onboarding` | Complete onboarding |
| GET | `/api/users/me/stats` | Get learning stats |
| DELETE | `/api/users/me` | Delete account |

### Words
| Method | Path | Description |
|---|---|---|
| GET | `/api/words` | Get word catalog (paginated, filterable) |
| GET | `/api/words/recommended` | Get personalized recommendations |
| GET | `/api/words/search` | Search words |
| GET | `/api/words/:id` | Get word detail |
| GET | `/api/words/me/list` | Get user's saved words |
| POST | `/api/words/me/:wordId/save` | Save a word |
| POST | `/api/words/me/:wordId/learned` | Mark as learned |
| POST | `/api/words/me/:wordId/difficult` | Mark as difficult |
| PATCH | `/api/words/me/:wordId/status` | Update word status |
| DELETE | `/api/words/me/:wordId` | Remove word |

### Practice
| Method | Path | Description |
|---|---|---|
| GET | `/api/practice/daily` | Get daily practice words |
| POST | `/api/practice/submit` | Submit practice session (server re-evaluates answers; ignores client `correct`) |

### AI
| Method | Path | Description |
|---|---|---|
| POST | `/api/ai/explain-word` | Get AI explanation |
| POST | `/api/ai/example-sentences` | Get example sentences |
| POST | `/api/ai/generate-quiz` | Generate quiz from saved words |
| POST | `/api/ai/generate-discover-quiz` | Generate quiz for new words |
| POST | `/api/ai/similar-words` | Get similar words |
| POST | `/api/ai/generate-words` | Generate and save new words |

### Learning Submission
- `GET /api/learning/continue` and `GET /api/learning/practice` return an `exerciseSessionId` bound to the authenticated user.
- `POST /api/learning/submit` should send that `exerciseSessionId` plus each answer's `questionHash` and `answer`.
- Open answers that cannot be safely graded return `pending_review`. They are stored in `LearningSession.answerRecords`, excluded from score math, skipped for word-memory penalties, and still count toward lesson completion progress.

## Features

- **Email + Google OAuth authentication** with JWT refresh token rotation
- **CEFR-aware vocabulary catalog** (A1–C2) with topics and search
- **Spaced repetition** tracking per user–word pair
- **AI features** (requires a DeepSeek key):
  - Level-appropriate word explanations
  - Contextual example sentences
  - AI-authored quiz generation with validated distractors
  - AI-authored Discover Quiz for new vocabulary
  - Similar word suggestions
- **Multilingual UI** — 10 locales (en, ru, es, fr, de, it, pt, ja, ko, zh), English default with per-user locale switching
- **Responsive layout** — optimized for mobile and desktop
- **Practice sessions** with accuracy-based word promotion

## Deployment Notes

For production:
1. Set `NODE_ENV=production`
2. Use a strong random `JWT_SECRET` and `JWT_REFRESH_SECRET`
3. Set `FRONTEND_URL` to your actual domain
4. Ensure `GOOGLE_CALLBACK_URL` matches your OAuth app's redirect URI
5. Use MongoDB Atlas or a managed MongoDB service
6. Set `MONGODB_URI` to your Atlas connection string
7. Configure CORS appropriately
8. Enable HTTPS (cookie `secure: true` is automatic in production)
